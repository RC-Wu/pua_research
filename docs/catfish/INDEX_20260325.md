# CatfishResearch Docs Index

This is the entrypoint for the CatfishResearch layer inside the integrated repository.

CatfishResearch is the repository identity. The Catfish Effect is the architectural concept: multi-project orchestration, parent-owned scoring, provider-aware routing, competing agent teams, and documentation as part of the runtime contract. Upstream ARIS remains the workflow lineage and historical provenance; the Codex-first fork history on `dev-intern-02` remains intentionally visible.

## Current Integrated Baseline In This Worktree

- Upstream ARIS mainline is preserved and explicitly tracked through the merge record.
- Catfish architecture docs are present under `docs/catfish/` and treat documentation as a runtime input, not an afterthought.
- Runtime core exists today in `tools/catfish_runtime.py`.
- Provider routing exists today in `tools/catfish_route_core.py`, `tools/catfish_route_eval.py`, and the router assets under `assets/router/`.
- The control center exists as a skeleton app under `apps/catfish-control-center/`.
- The `cc-switch` bridge exists today in `tools/cc_switch_bridge.py`.
- Remote subagent execution remains available through [`skills/skills-codex/remote-codex-subagents/SKILL.md`](../../skills/skills-codex/remote-codex-subagents/SKILL.md), with supporting heartbeat and final-summary scaffolds adjacent to it.

## Provenance Rules

- Do not describe CatfishResearch as a clean-slate replacement for ARIS. It is an integrated, documentation-first rebrand of the combined repository state.
- Keep upstream ARIS attribution explicit when touching top-level docs, merge notes, or workflow descriptions.
- Keep the Codex-first fork history explicit when touching provider routing, remote subagents, or control-plane notes.
